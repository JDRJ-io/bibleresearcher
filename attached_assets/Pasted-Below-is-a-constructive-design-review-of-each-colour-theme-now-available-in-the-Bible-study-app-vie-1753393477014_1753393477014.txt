### Cross‑theme, high‑impact improvements

| Area                    | Recommendation                                                                                                                                                                                                      |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Typography**          | Adopt a two‑font system: “**Inter**” (UI & verse text) and “**Source Serif Pro**” (book titles / references). 1.5 em line‑height in verse column ensures Old‑Testament poetry stanzas don’t crowd.                  |
| **Grid & spacing**      | Reduce table border‑radius to 4 px (currently square) for modern but restrained look. Provide `row-gap: 8px` with transparent background rather than heavy outlines for light themes.                               |
| **Focus/hover states**  | Introduce an **animated underline** or **soft box‑shadow** instead of colour change for links; it works in every palette without recalibrating colours.                                                             |
| **Global Variables**    | Move every colour to `/styles/tokens.css` as CSS custom properties (`--surface‑1`, `--text‑primary`, `--accent`, `--grid`) and use HSL for easy lightness tuning. This makes future palette tweaks a 10‑minute job. |
| **Accessibility audit** | Run `npm i --save-dev @deque/axe-core` and integrate a one‑time automated test (`axe.run`) in dev build; fix any contrast violations flagged (especially Rainbow, Cyber, Forest).                                   |
| **Eye‑strain mode**     | Add a per‑theme *“study dimmer”* slider that multiplies background `hsla` lightness by 0.8 → 1.2. Users can fine‑tune for their monitor and ambient light without code changes.                                     |

### How to apply the tweaks quickly

1. **Tokenise colours** → create a mapping object in `themeProvider.tsx` so each theme inherits *base* + *accent* tokens and overrides only what it needs.
2. **Utility classes** → in `index.css` define `.text-primary`, `.bg-surface`, `.grid-border` etc. and reference tokens.
3. **Palette testing** → add a hidden route `/dev/contrast?theme=forest` that mounts Axe‑core and prints failing elements to console.
4. **Design QA checklist** (one‑time for every new palette):

   * 4.5:1 contrast for normal text ✔︎
   * Headings heavier than body ✔︎
   * No more than **2 accent hues** ✔︎
   * Hover state discoverable within 0.2 s ✔︎

Following the small, targeted colour & contrast adjustments above will give every palette a consistent, scholarly character while still letting users express personal preference.
